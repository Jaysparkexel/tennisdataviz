Write Up for Assignment 2

Data Pre-processing

I decided to go with match tree visualization after searching through D3.js examples. So, I looked at how it was implemented and founded that the biggest challenge was data formatting which was required for making any kind of game tree. I have to follow the special structure format of parent and child in JSON. I did data pre-processing in python using pandas. I wrote up a script to generate desired JSON format for every year data was available. During data pre-processing, I found out that some of the matches were missing from the data. Data for 11 matches were missing. three of these matches were a walkover match. Most missing matches were from the first round and some of them were from the third round. For all the missing matches, I added only fields that were required for my visualization. I found the match data on Wikipedia. I have provided one updated CSV file and JSON file in a required format for 11 years. I also have provided a Python program named pre-processing.py that I used to generate these files.

Visualization Design and Story

From the match tree, we can easily observe who played against whom in the tournament. While hovering over match tree three players are highlighted with different color, which implies a match between players and the winner among them. The name of the player is shrunk to 3 letters so It can be fitted in match tree. The full player name is displayed when hovering over the match. Each circle is round of the tournament. The middle circle is used for the final match. The outer most circle is used for all the first round matches of the tournament. The round name is provided on each arc. As we move from the outer arc to the inner arc number of player decreases as players disqualify and the rounds move one step closer to the final round. By hovering over a match, the final score of the match and who won is displayed. The color schematic is chosen from tennis inspired colors. On the left side year navigation bar is given. The player can choose the year for which they want to see the game tree.

Interaction and Color Scheme

This visualization is interactive. The user can choose the game tree from 11 years of the tournament. In every game tree, the user can interact with the tree and get different information. From observing this visualization user can follow the player in the whole tournament. Like against whom he played, he won or lose against that player, what was the score for that match and how far player reached in the tournament. So, this one game tree shrinks down and communicate so much information of one year to the user. Instead of traditional drop-down for choosing year I have provided the interactive list to select a year. If there are more than one name with same generated abbreviation than there will be conflict when highlighting the match triplet.
